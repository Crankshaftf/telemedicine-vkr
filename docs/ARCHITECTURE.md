# Архитектура системы «МедКоннект»

Краткое описание для раздела «Реализация» ВКР.

## 1. Назначение

Демонстрационный прототип телемедицинской платформы для медицинской организации.  
Позволяет пациенту записаться на дистанционную консультацию, пройти чат с врачом и получить заключение.

## 2. Архитектура

Клиент-серверная модель:

```
┌─────────────────┐     REST/JSON      ┌──────────────────┐
│ Android-приложение│ ◄──────────────► │  FastAPI-сервер  │
│  (пациент)       │     JWT-токен     │                  │
└─────────────────┘                    │  ┌────────────┐  │
                                       │  │ PostgreSQL │  │
┌─────────────────┐     REST/JSON      │  └────────────┘  │
│  Веб-панель     │ ◄──────────────► │  Файлы: uploads/ │
│ (врач/админ)    │                    └──────────────────┘
└─────────────────┘
```

## 3. Компоненты

| Компонент | Технологии | Назначение |
|-----------|------------|------------|
| Android-клиент | Kotlin, Jetpack Compose, MVVM, Retrofit, DataStore | UI пациента |
| Backend API | Python, FastAPI, SQLAlchemy, JWT | Бизнес-логика, авторизация |
| БД | PostgreSQL 16 | Хранение данных |
| Веб-панель | HTML, CSS, JavaScript | Интерфейс врача и администратора |
| Файловое хранилище | Локальная папка `uploads/` | Медицинские документы |

## 4. Роли пользователей

- **Пациент** — Android-приложение
- **Врач** — веб-панель (записи, чат, заключение)
- **Администратор** — веб-панель (расписание, статусы, отчёты)

## 5. Модель данных (основные таблицы)

- `users` — учётные записи, роли
- `patients`, `doctors` — профили
- `schedule_slots` — расписание (free / busy / blocked)
- `appointments` — заявки на консультацию
- `consultations`, `messages` — чат и заключение
- `medical_files` — прикреплённые файлы
- `notifications` — in-app уведомления

## 6. Безопасность

- HTTPS (в production)
- Хэширование паролей (bcrypt)
- JWT-авторизация, срок 24 ч
- Разграничение прав по ролям
- Пациент видит только свои данные

## 7. Контрольный сценарий

1. Пациент (`user@test.ru`) входит в Android-приложение
2. Выбирает врача и свободное время
3. Создаёт заявку с жалобами и файлом
4. Врач (`doctor@test.ru`) в веб-панели открывает запись
5. Ведёт чат и сохраняет заключение
6. Пациент видит результат в истории обращений

## 8. Перспективы развития

- Push-уведомления (Firebase Cloud Messaging)
- Видеосвязь (WebRTC)
- Интеграция с медицинскими информационными системами
- Оплата консультаций
